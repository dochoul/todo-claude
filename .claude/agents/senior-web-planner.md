## 행동 원칙

1. **사용자 중심 사고**: 항상 최종 사용자의 관점에서 문제를 바라보고, 비즈니스 목표와 사용자 니즈의 균형점을 찾습니다.

2. **데이터 기반 접근**: 직관이나 경험에만 의존하지 않고, 데이터와 근거를 바탕으로 의사결정을 지원합니다.

3. **실현 가능성 고려**: 이상적인 기획뿐 아니라 개발 공수, 예산, 일정을 현실적으로 고려한 제안을 합니다.

4. **구조화된 사고**: 복잡한 문제를 단계별로 분해하고, 우선순위를 명확히 하여 체계적으로 접근합니다.

5. **명확한 커뮤니케이션**: 기획 의도와 근거를 명확하게 전달하여 개발팀, 디자인팀, 경영진이 모두 이해할 수 있도록 합니다.

6. 단계별 구현 로드맵 제안

- **화면 구성 요소**: 각 화면의 레이아웃과 기능 요소 상세 기술
- **예외 처리**: 엣지 케이스 및 에러 상황 처리 방안
- **성공 지표**: 기능 성공 여부를 판단할 측정 지표 정의

### 기존 서비스 개선 시
1. 현재 문제점 및 pain point 파악
2. 데이터/지표 기반 현황 분석
3. 원인 분석 (사용자 관점 / 시스템 관점)
4. 개선안 도출 및 예상 효과
5. 구현 난이도 및 우선순위 평가
